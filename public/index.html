FishingLog

A simple PLATO client for fishing sessions

Captain's Mate logs trips to a PLATO server and queries them back by species, recency, or location, with a lightweight natural-language question interface on top. It is a sibling logging-and-recall tool to DeckBoss, not a competitor or rebrand.

What This Is

fishinglog-agent ("Captain's Mate") is a real Python package, published on PyPI (Development Status: Alpha). It is a client for a PLATO server: it POSTs session records, GETs them back, and filters by species, recency, and a rough haversine-style radius. A simple natural-language interface sits on top of that filtering — useful for recall, not statistical analysis.

Quick Start

pip install fishinglog-agent

from fishinglog_agent import FishingLogAgent

agent = FishingLogAgent()
agent.log_session(
    latitude=41.5,
    longitude=-71.3,
    depth_meters=45,
    species="tuna",
    catch_count=12,
    notes="Good action on the tide change",
)

Honesty note: fishinglog-agent ("Captain's Mate") is a real, published PyPI package. There is no slick product wrapped around it yet, but the code is installable today and the logging and query logic is real.

PLATO note: this is a client, not a standalone tool. Logging or querying anything beyond import requires a running PLATO server at the plato_url you pass in.

Forward-looking · 2026-07-08

Where this is going

Everything above is what fishinglog-agent does today. Separately — and not yet built — fishinglog.ai is planned to become one of eight domain-specific skins on top of a shared voice-capture-and-storage core called activelog.ai. That core is the real work; this skin is a later consequence of it, not a thing being built on this page.

The shared core, in the shape it's planned in:

01 Pick a storage location

A local folder, your own Cloudflare account, Google Drive, a database — yours, not operated by this project.

02 Dictate into a structured log

Activate the mic and talk; speech becomes structured log entries, not a raw transcript.

03 Organize it

Pause and wake-word detection segment and label the capture as you go.

04 Ask it questions

A wake-word chat panel — bring your own model key — reads back and queries what you logged.

Status, plainly: this is a real, dated plan (2026-07-08), not yet built. The eight domain-specific skins — this page included — do not start until the activelog.ai core works for someone who didn't build it, verified the same way the org verifies anything else. When that gate opens, this page is rebuilt on top of the real core — not before. Track actual progress at activelog.ai and at the org's honest domain registry, purplepincher.org/#domains. No timeline is implied here, because none exists.
